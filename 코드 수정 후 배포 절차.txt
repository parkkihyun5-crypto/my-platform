# 코드 수정 후 표준 배포 절차
## 1단계. 프로젝트 폴더로 이동
PowerShell에서는 `cd /d`를 쓰지 않습니다. 
아래처럼 입력합니다.

```powershell

D:
cd D:\projects\my-platform


또는 한 줄로는 이렇게 입력합니다.

```powershell
Set-Location D:\projects\my-platform

---

## 2단계. 수정 상태 확인
```powershell
git status


여기서 수정된 파일이 표시됩니다.

예를 들어:

```txt
modified: app/social-enterprise/page.tsx
modified: app/branding/page.tsx
modified: app/public-interest-foundation/page.tsx


이렇게 나오면 아직 GitHub에 올라가지 않은 로컬 수정 상태입니다.

---

## 3단계. 빌드 테스트
배포 전 반드시 아래 명령을 실행합니다.

```powershell
npm run build

빌드가 통과해야 배포해도 됩니다.
성공하면 대략 아래처럼 나옵니다.

```txt
Compiled successfully

또는 Next.js 빌드가 끝까지 완료됩니다.
실패하면 배포하지 말고 오류 파일을 먼저 수정해야 합니다.
---

## 4단계. 전체 변경사항 배포할 경우
현재 요청하신 방식처럼 **my-platform 전체 변경사항을 한 번에 배포**하려면 `deploy.bat`를 더블클릭하면 됩니다.

이때 실행되는 흐름은 다음과 같습니다.
```txt
npm install
↓
npm run build
↓
git add -A
↓
git commit -m "full deploy my-platform 날짜_시간"
↓
git push origin 현재브랜치
↓
Vercel 자동 배포
```
즉, `deploy.bat`는 **빌드가 성공한 경우에만 GitHub에 올리고**, GitHub에 올라가면 Vercel이 자동 배포합니다.
---

# 가장 안전한 실무 절차
앞으로는 아래 순서로 하시면 됩니다.

```powershell
D:
cd D:\projects\my-platform
git status
npm run build

빌드가 성공하면:

```txt
deploy.bat 더블클릭

빌드가 실패하면:

```txt
오류 메시지 확인 → 해당 파일 수정 → 다시 npm run build

---
# 주의할 점
`deploy.bat` 안에 `git add -A`가 들어 있으므로, 현재 프로젝트 안의 모든 변경 파일이 함께 배포됩니다.

따라서 아래처럼 여러 파일이 동시에 수정되어 있으면:
```txt
M app/social-enterprise/page.tsx
M app/branding/page.tsx
M app/public-interest-foundation/page.tsx
M public/images/consultant-profile.psd

`deploy.bat` 실행 시 전부 커밋 대상에 포함됩니다.

---

# 특정 파일만 배포하고 싶을 때
예를 들어 사회적기업 페이지 하나만 올리고 싶다면 `deploy.bat`를 쓰지 말고 수동으로 합니다.

```powershell
D:
cd D:\projects\my-platform
npm run build
git add app/social-enterprise/page.tsx
git commit -m "Update social enterprise service packages"
git push origin main
```

현재 브랜치가 `main`이 아니라면 아래로 확인합니다.
```powershell
git branch --show-current

확인된 브랜치명으로 push합니다.
```powershell
git push origin 브랜치명

---

# 전체 배포와 부분 배포의 차이
| 방식    	 	| 명령               	 	| 용도                   	 	|
| ------		| -----------------	 	| ----------------------	 	|
| 전체 배포  	| `deploy.bat` 더블클릭 	| my-platform 전체 변경사항 배포 	|
| 부분 배포 	| `git add 특정파일`   	| 특정 파일만 골라서 배포          	|
| 빌드 확인만 	| `npm run build`   		| 오류 확인                  		|
| 상태 확인  	| `git status`      		| 어떤 파일이 수정됐는지 확인        	|

---

# 앞으로의 권장 기준
## 평소에는 이 방식

```txt
1. 코드 수정
2. npm run build
3. 오류 없으면 deploy.bat 더블클릭

## 중요한 파일만 따로 올릴 때
```txt
1. 코드 수정
2. npm run build
3. git add 특정파일
4. git commit
5. git push

## 여러 파일을 의도적으로 모두 올릴 때
```txt
1. 코드 수정
2. npm run build
3. deploy.bat 더블클릭

---
